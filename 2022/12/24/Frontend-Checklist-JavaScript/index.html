JavaScript Checklist: Variables & Types

Gloria Su
2022-12-24

1. 语言类型

Undefined, Null, Boolean, Number, String, Object, Symbol, BigInt.

2. 变量在内存中的存储形式

  • 堆（heap）
    堆内存，动态分配内存，内存大小不一，不是自动释放。
    队列优先, 先进先出，方便存储和开辟内存空间。如果存储在栈中，将会影响程序运行的性能；

  • 栈（stack）
    栈内存，自动分配相对固定大小的内存空间，并由系统自动释放。基本类型存储在栈内存的好处就是，内存可以及时的得到回收。
    线性结构，后进先出，便于管理。

  • 值类型：主要是指基本类型，即 number，string，boolean，undefined，null，symbol。保存在栈内存中的简单数据段，它们的值都有固定的大小，保存在栈空间，通过值复制的方式赋值和传递值。

  • 引用类型：除上述值类型外的对象类型。保存在堆内存中的对象，值大小不固定，栈内存中存放的该对象的访问地址指向堆内存中的对象，JavaScript 不允许直接访问堆内存中的位置，总是通过引用复制的方式赋值和传递值。

3. Symbol 类型

es6 中新引入的类型，表示独一无二的值，最大的用法是用来定义对象的唯一属性名。不能使用 new 命令，否则会报错。这是因为生成的 Symbol 是一个原始类型的值，不是对象。
应用场景：

  1. 使用 Symbol 来作为对象属性名(key)；
  2. 使用 Symbol 来替代常量；
  3. 使用 Symbol 定义类的私有属性/方法。
  • symbol 作为属性名，是公有属性，无法被 Object.keys() 循环获取，可以通过 Reflect.ownKeys() 和 Object.getOwnPropertySymbols() 取到。
  • instanceof 的结果为 false，typeof 的结果为 “symbol”。

4. BigInt 类型

es10 新增的基本类型，可以表示任意大小的整数。

  • 创建 BigInt，只需在整数的末尾追加 n 即可, 可以调用 BigInt() 构造函数。
  • 不能使用严格相等运算符将 BigInt 与常规数字进行比较，因为它们的类型不同。
  • 除一元加号(+)运算符外，所有算术运算符都可用于 BigInt。
  • 除法(/)运算符的结果会自动向下舍入到最接近的整数。
  • 隐式类型转换可能丢失信息，所以不允许在 BigInt 和 Number 之间进行混合操作。
  • 关系运算符不遵循此规则。

5. 基本类型 装箱拆箱

装箱：基本类型 => 对象类型

  • 显示装箱：通过内置对象 String、Boolean、Number 等可以对基本类型进行显示装箱
  • 隐式装箱：当读取一个基本类型值时，后台会为该基本类型创建一个对应的基本类型包装对象。在这个基本类型上调用方法，实际上是在这个基本类型包装对象上调用方法。这个基本类型包装对象是临时的，它只存在于方法调用那一行代码的的瞬间，一旦调用完毕就立即销毁。

拆箱：对象类型 => 基本类型
拆箱操作可以通过一些操作符来实现，但实际调用的是toString()和valueOf()。

6. 浅拷贝 深拷贝

  • 浅拷贝
    Object.assign() 只复制了第一层属性，复制对象是基本类型。
  • 深拷贝
    对属性中所有引用类型的值，遍历到是基本类型的值为止，利用递归来实现深拷贝。

7. null 和 undefined 区别

  1. Number 转换的值不同，Number(null) = 0, Number(undefined) = NaN
  2. null 表示一个值被定义了，但是这个值是空值，typeof 得到 “object”。
  3. undefined 表示缺少值，即此处应该有值，但是还没有定义。

8. 判断数据类型的方式

  1. typeof：返回的结果包括 number、boolean、string、object、undefined、function 6 种数据类型。typeof(null) 是 object，原因 JavaScript 中的数据在底层是以二进制存储，比如 null 所有存储值都是 0，但是底层的判断机制，只要前三位为 0，就会判定为 object。
  • 优点：判断基本类型较为准确；
  • 缺点：无法精准的判断引用类型，null error() date() array() 都返回 object
  1. instanceof：其内部运行机制是判断在其原型链中能否找到该类型的原型。检测 constructor.prototype 是否存在于参数 object 的原型链上，是则返回 true，不是则返回 false。instanceof 只能用来判断两个对象是否属于实例关系，而不能判断一个对象实例具体属于哪种类型。
  • 优点：判断引用类型较为准确；
  • 缺点：无法对基本类型作出精准的判断；它限定了我们的数据类型必须是用 new 出来的才能准确的判断。
  1. constructor：每个函数的定义，都会生成一个constructor。基本类型会执行隐时的执行装箱操作，创造一个构造函数的实例。
  • 优点：对基本类型和引入类型都可以判断；
  • 缺点：无法判断 null 和 undefined，而且 constructor 是可以修改的，会导致检查结果不准确。
  1. Object.prototype.toString.call()：所有原始数据类型都是能判断的。
  • 缺点：Object.prototype.toString 本身也可能被修改，会导致检查结果不准确。

9. 隐式类型转换

  • 发生场景：if、while、switch 判断语句中会隐式转化为布尔值；布尔操作符、相等操作符等。
  • 如何避免：使用类型检查来防止转换导致的结果超出预知。
  • 巧妙应用：布尔值的转换经常用在条件语句中。

10. 小数精度丢失

  • 出现精度丢失的原因：在 JS 中采用双精度存储，占用 64bit。1 位用来表示符号位，11 位用来表示指数，52 位用来表示尾数。
  • JS 中的 Number 类型只能安全地表示 Number.MAX_SAFE_INTEGER = -(2^53-1) 和 Number.MIN_SAFE_INTEGER = 2^53-1 之间的整数，任何超出此范围的整数值都可能失去精度。
  • 超出安全整数限制可以使用 BigInt。
  • 对于浮点数的处理，为了避免精度丢失，通常情会转化为整数处理，然后通过算数运算符转化为浮点数。
